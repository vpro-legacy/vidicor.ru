Система удаленного мониторинга распределенной сети датчиков.
Техническое описание

Система удаленного мониторинга распределенной сети датчиков состоит из связанных через сеть GSM некоторого множества модулей регистрации состояния датчиков, центрального сервера, и мобильного терминала оповещения, а также рабочих мест операторов, связанных с центральным сервером через локальную сеть или Интернет.

К модулям регистрации состояния датчиков подключаются датчики (напр., о завершении товара в диспенсере). При срабатывании/отпускании какого-либо из датчиков модуль оповещения отправляет сообщение на центральный сервер.

Центральный сервер обеспечивает наглядный человеко-компьютерный интерфейс с сотрудником-оператором.

На экране центрального сервера обеспечивается отображение текущего состояния датчиков. В случае срабатывания датчика, если состояние срабатывания длится более заданного времени, то на экран выдается сообщение, в котором указывается, какой датчик сработал. В случае, если состояние срабатывания длится более заданного времени, центральный сервер отправляет сообщение на мобильный терминал оповещения, которым является обычный сотовый телефон, способный принимать SMS сообщения.

Проводить мониторинг возможно не только с центрального сервера, но с любого компьютера из любой точки, имеющей выход в Интернет при условии подключения к Интернет центрального сервера системы и этого компьютера.

Структура системы:

Модуль регистрации состояния датчиков:

Система GSM сигнализации предназначена для удаленного мониторинга распределенной сети датчиков через сеть GSM. Проводить мониторинг возможно из любой точки, имеющей выход в Интернет при условии подключения к Интернет центрального сервера системы.

Система позволяет:

- проводить мониторинг как отдельных модулей регистрации состояния датчиков, так и их групп, и всей системы; при этом количество модулей регистрации состояния датчиков не ограничено;

- отправлять SMS-сообщения о срабатывании датчиков в формате, задаваемом пользователем, на сотовый телефон с предустановленным номером;

- генерировать отчет о состоянии системы на текущий момент времени;

- вести статистику событий.

Подготовка к работе модуля регистрации состояния датчиков

Перед инсталляцией модуля регистрации состояния датчиков следует инициализировать SIM-карту. Для этого можно использовать произвольный сотовый телефон. В первую ячейку адресной книги SIM-карты следует занести телефонный номер GSM-модуля центрального сервера.

Проинициализированную таким образом SIM-карту следует установить в модуль регистрации состояния датчиков.

Подключение датчиков осуществляется к разъемам на передней панели модуля регистрации состояния датчиков. В разъеме задействованы 1-й и 3-й контакты. При замыкании (или размыкании) этих контактов на центральный сервер отправляется сообщение о замыкании (или размыкании) контактов датчика.

Для включения системы следует подключить блок питания к электросети. Затем – с помощью какого-либо жесткого стержня диаметром 2-3 мм через отверстие в корпусе с маркировкой «1» нажать и удерживать в течение 3-х секунд кнопку включения питания модуля. После этого с помощью того же стержня нажать кнопку запуска через отверстие с маркировкой «2».

Инсталляция и настройка центрального сервера системы оповещения о состоянии распределенной сети датчиков

Сервис центрального сервера служит для синхронизации модуля GSM и веб-интерфейса сервера.

Файлы сервиса центрального сервера:

- tttsvc.exe – исполняемый файл сервиса.

- config.ini – конфигурационный файл сервиса.

- install.bat – файл скрипт, для установки сервиса в ОС.

- start.bat – файл скрипт, для запуска сервиса.

- stop.bat -  файл скрип, для остановки сервиса.

Порядок подключения и настройки сервиса серверной части.

- Подключите GSM-модуль к серверу.

- Установите дополнительное ПО для поддержки GSM-модема сервером, если это необходимо.

- В конфигурационном файле config.ini сервиса пропишите параметром “COM” номер COM порта на который установился (подключен) модем телефона в системе.

- Проверьте все пути в конфигурационном файле указанные в нем пути; они должны совпадать с фактическими, куда было установлено ПО сервера

- Запустите install.bat (его необходимо запускать только один раз).

- Запустите сервис  (после перезагрузки системы повторный запуск сервиса не требуется).

Внимание!  Для работы серверной части необходим ключ HASP, поставляемый вместе с ПО.

Управление центральным сервером и его настройка

Настройка номера мобильного терминала оповещения

На данный номер будет отправляться сообщение о срабатывании датчика в каком-либо из модулей регистрации состояния датчиков в содержательных терминах.

В формате, задающемся при вводе:

[ Название группы] [ Название диспенсера] [Название типа сигарет, которые отсутствуют] [type x – номер сработавшего датчика]

(О данных настройках см. далее)

Пример:

“Yekaterinburg, IMM, Dispenser 10,  Sobranie empty [type 2]”

Если отправлять СМС не нужно, то необходимо снять галочку “Send SMS?” и нажать  кнопку “Set”  

Если необходимо сменить номер курьера, то нужно ввести новый номер в формате [код станы][номер мобильного телефона] Перед номером необходимо убирать “+” в поле и нажать кнопку “Set”

Добавление нового модуля регистрации состояния датчиков в систему

Для добавления модуля регистрации состояния датчиков в систему необходимо заполнить следующие поля:

Group: [ Number ] – Номер группы в которую будет входить, диспенсер, например  “0”.

Group Name [ Text ] – название группы. Если вы добавляете диспенсер в уже существующую  группу, и не хотите изменять ее название оставьте это поле пустым.

Number [ Number ] – Номер сотового телефона, установленного в диспенсере,

с которого будут приходить сообщения о показаниях датчика. (Формат см. номер курьера).

Descriptions [ Text ] – Текст, заменяющий номер диспенсера; используется при отображении. например “Dispenser 10”.

Далее необходимо заполнить поля с данными о типах сигарет, размещаемых в диспенсере с 1 по 8, например, просто указав названия типов сигарет напр. “Sobranie”.

После того, как все поля заполнены, нажмите кнопку  .

Новый диспенсер должен появиться в списке

Удаление модуля регистрации состояния датчиков

Для того, чтобы удалить модуль регистрации состояния датчиков, необходимо поставить галочку напротив одного или нескольких диспенсеров, которых вы хотите удалить.

Затем нажмите кнопку “Delete selection”
